# Prompt: Lead Management CRM - Neon Dashboard

> **Context**: System for coordinating leads generated by Neon students, following modern CRM design best practices (2025) and integrating with the Neon Dashboard ecosystem.

## ğŸ“‹ Overview

**Objective**: Build a complete and well-structured CRM for lead management, focused on efficiency, pipeline visualization, and productivity for Neon mentees/students.

**CRM Design Principles 2025** (research-based):

- **Visual Pipeline**: Kanban board for clear lead flow visibility
- **Touch Targets**: 48px+ for primary CTAs (Fitts' Law)
- **Progressive Disclosure**: Advanced filters hidden until needed (Hick's Law)
- **Contextual Insights**: Data transformed into actionable actions
- **Role-Based Views**: Interface adapted to user type (mentor/student)

export const leads = pgTable("leads", {
  id: serial("id").primaryKey(),
  user_id: integer("user_id").references(() => users.id), // mentee owner
  nome: text("nome").notNull(),
  email: text("email").notNull(),
  telefone: text("telefone"),
  empresa: text("empresa"),

  status: text("status").notNull(), // pipeline stage
  valor_estimado: integer("valor_estimado"),
  data_criacao: timestamp("data_criacao").defaultNow(),

  tags: text("tags").array(),

export const interacoes = pgTable("interacoes", {
  id: serial("id").primaryKey(),

  data: timestamp("data").defaultNow(),
  user_id: integer("user_id").references(() => users.id),
